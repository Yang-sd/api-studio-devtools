## 我们收集或处理的数据

API Studio DevTools 是开发者工具扩展，主要在浏览器 DevTools 面板中运行。为了提供调试能力，扩展可能在用户本地浏览器中处理以下数据：

- 当前调试页面产生的网络请求信息，例如 URL、Method、Status、Headers、请求体、响应体、耗时等。
- 用户手动保存的 Replay 请求历史。
- 用户创建或导入的 Mock 规则。
- 用户调试时查看的 Cookie 和 Set-Cookie 信息。
- 用户配置的分组、命中计数和主题偏好。

## 数据保存位置

上述数据默认保存在用户浏览器本地，例如 `chrome.storage.local` 或页面本地存储。扩展本身不包含后端服务器，也不会主动把这些数据上传到第三方服务器。

## 数据用途

这些数据仅用于以下本地开发调试功能：

- 展示当前页面的 API 请求详情。
- 重放用户选择的请求。
- 根据用户创建的规则返回本地 Mock 响应。
- 辅助分析埋点请求、Cookie 和登录态问题。
- 保存用户界面偏好，例如主题模式和面板布局。

## 数据共享

API Studio DevTools 不出售用户数据，不与第三方共享用户数据，也不将用户的请求内容、Cookie、Mock 规则或 Replay 历史上传到外部服务。

## 权限说明

扩展使用 `storage` 权限保存本地调试数据。扩展使用 `<all_urls>` host permissions，是因为开发者需要在不同域名、测试环境、本地服务和线上页面中调试 API 请求。该权限仅用于当前浏览器调试场景，不会修改系统 hosts 文件。

## 用户控制

用户可以在扩展面板中删除 Mock 规则、Replay 历史、Cookie 记录或埋点命中记录。用户也可以通过浏览器扩展管理页面移除扩展，从而删除扩展相关本地数据。

## 远程代码

API Studio DevTools 不从远程服务器加载或执行扩展业务代码。扩展所需脚本随扩展包一并提供。

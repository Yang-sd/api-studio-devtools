# API Studio DevTools

API Studio DevTools is a browser DevTools extension for capturing, replaying, mocking, and inspecting API requests during frontend development.

## Features

- **Network**: capture requests from the current page and inspect URL, method, status, headers, cookies, and response body.
- **Replay**: save API requests, organize them by group, rename them, and resend them from the DevTools panel.
- **Mock**: create local mock rules from captured requests and toggle mock behavior with a master switch.
- **Throttle**: simulate basic weak-network behavior for delay, jitter, upload speed, and download speed.
- **Beacon**: monitor analytics/reporting APIs and inspect parsed payload fields.
- **Cookies**: collect and inspect request cookies and response `Set-Cookie` values.
- **Chrome and Firefox**: includes separate manifests for Chromium-based browsers and Firefox.

## Load In Chrome

1. Open `chrome://extensions`.
2. Enable **Developer mode**.
3. Click **Load unpacked**.
4. Select this repository folder.
5. Open DevTools and choose the **API Studio** tab.

## Load In Firefox

1. Open `about:debugging#/runtime/this-firefox`.
2. Click **Load Temporary Add-on**.
3. Select `manifest.firefox.json` from this repository.
4. Open DevTools and choose the **API Studio** tab.

## Test App

The `test/` directory contains a small Flask app for trying API capture, replay, mock, throttle, beacon, and cookie workflows.

Run locally:

```bash
cd test
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Open:

```text
http://localhost:5000
```

Run with Docker:

```bash
cd test
docker build -t api-studio-test-app .
docker run --rm -p 5000:5000 api-studio-test-app
```

## Notes

- The extension does not modify system hosts files.
- Weak-network simulation is browser-extension-level simulation, not OS-level packet loss.
- Runtime files generated by the test app, such as SQLite databases, logs, PID files, and virtual environments, are ignored by Git.
